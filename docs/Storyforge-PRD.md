이 PRD를 보고 개발하는 분(또는 AI)은 다음의 비교를 염두에 두고 진행해 주세요:

| --- | --- |

| 터미널/콘솔 | AI 대화창 |
| 사이드바 확장 | 세계관 패널 |

## 설계 원칙

이 철학을 바탕으로 다음 원칙을 따라주세요:

1. **파일 탐색기 같은 직관성:** 트리 뷰는 윈도우 탐색기처럼 익숙해야 함
2. **에디터 중심:** 화면의 가장 넓은 영역은 항상 글쓰기 공간
3. **컨텍스트 유지:** AI는 현재 작품의 설정, 줄거리, 인물 정보를 항상 인지
4. **확장성:** 패널은 숨기거나 확장할 수 있어야 함
5. **키보드 중심:** 파워 유저를 위한 단축키 지원 고려
6. **오프라인 우선:** 로컬 저장이 기본, 클라우드는 동기화 용도

# 2. 문제 정의 및 해결 방안

## 2.1 현재 창작자들의 문제

**파편화된 워크플로우**
- 아이디어 정리: 노션, 에버노트
- 구조 설계: 스크리브너, 엑셀
- 집필 및 제출: 아래아한글(HWP)

**기존 툴의 한계**
- HWP: 집필용으로는 무겁고, 세계관 정리나 순서 변경이 어려움
- 스크리브너: 학습 곡선이 높고, 한국어 환경에 최적화되어 있지 않음
- 기존 AI 도구: 단순 글쓰기 보조만 제공, 맥락 관리 불가

## 2.2 Storyforge의 해결 방안

- **통합 환경:** 구상, 설정, 집필을 하나의 화면에서 처리
- **직관적 UI:** 별도 학습 없이 바로 사용 가능
- **현실적 호환성:** HWP로의 복사-붙여넣기 최적화
- **AI 지능:** 맥락을 유지하며 창작 지원

# 3. 타겟 사용자

## 3.1 메인 타겟

| 구분 | 설명 |
| --- | --- |
| 웹소설 작가 | 연재 주기가 빠르고 세계관 관리가 중요 |
| 드라마/시나리오 작가 | 씬 구조 관리가 필요 |
| 웹툰 스토리 작가 | 에피소드별 플롯 관리 필요 |

## 3.2 사용자 수준

입문자부터 전업 프로 작가까지 **전 수준** 포함

## 3.3 지원 장르

**전 장르** 지원 (판타지, 로맨스, 현대물, 무협, 스릴러, SF 등)

## 3.4 주요 사용 시나리오

- 아이디어는 있지만 글쓰기가 어려운 사람이 도구의 도움으로 완성
- 이미 글을 쓰는 작가가 생산성을 높이기 위해 활용
- 복잡한 세계관을 한눈에 보며 집필하고 싶은 작가

# 5. 화면 구성 (Layout)

## 5.1 전체 레이아웃

**3열 구성** 유지

| 위치 | 역할 | 설명 |
| --- | --- | --- |
| 좌측 패널 | 탭 전환 방식 | 구조 탭 / 설정 탭 전환 |
| 가운데 | 에디터 영역 | 직접 수정 가능한 집필 공간 |
| 우측 패널 | AI 대화창 | AI와의 실시간 대화 |

## 5.2 좌측 패널 상세 (탭 전환 방식)

### 탭 1: 구조 탭 (Structure)

| 구성 | 설명 |
| --- | --- |
| 트리 뷰 | 권-장-절 또는 회차-씬 구조 |
| 드래그 앤 드롭 | 마우스로 순서 즉시 변경 |
| 상태 표시 | 구상 중 / 집필 중 / 탈고 / 업로드 완료 |
| 새 회차 추가 | + 버튼으로 빠른 추가 |

### 탭 2: 설정 탭 (Settings)

| 단 | 내용 |
| --- | --- |
| 1단 | 진행상황 (현재 권/화, 총 글자수) |
| 2단 | 인물목록 (등장인물 및 현재 상태) |
| 3단 | 최근요약 (AI 참조용 요약) |
| 4단 | 버튼 영역 |

**4단 버튼 상세:**
- [줄거리] 전체 스토리 시놉시스
- [관계도] 인물 관계 다이어그램
- [복선관리] 떡밥 회수 여부 체크
- [세계관] 지명, 조직, 아이템 등

## 5.3 시각적 레이아웃


# 6. 핵심 기능

## 6.1 구조 관리 (Structure Management)

| 기능 | 설명 |
| --- | --- |
| 트리 뷰 | 윈도우 탐색기 형태로 권-장-절 구조 관리 |
| 드래그 앤 드롭 | 마우스로 씬(Scene) 순서 즉시 변경 |
| 상태 표시 | 구상 중 / 집필 중 / 탈고 / 업로드 완료 아이콘 |
| 템플릿 | 소설 / 시나리오 템플릿 선택 |

## 6.2 에디터 및 집필 환경 (Writing Experience)

| 기능 | 설명 |
| --- | --- |
| 집중 모드 | 사이드바와 메뉴를 숨기고 흰 화면에 글자만 |
| 화면 분할 | 좌측 원고 + 우측 참고자료 동시 보기 |
| 자동 저장 | 타이핑 멈춤 1~2초 후 자동 저장 |
| 버전 히스토리 | 이전 버전으로 되돌리기 가능 |
| 글자수 표시 | 실시간 글자수 카운터 |

## 6.3 세계관 아카이브 (World Building)

| 기능 | 설명 |
| --- | --- |
| 카드형 DB | 인물/장소/아이템을 카드로 생성, 이미지 첨부 가능 |
| 퀵 메모 | 에디터 우측에서 언제든 메모 작성 및 검색 |
| 링크 기능 | 에디터 내 이름 클릭 시 캐릭터 카드 팝업 |
| 관계도 | 텍스트 기반 + 시각적 다이어그램 동시 제공 |

**인물 카드 포함 항목:**
- 기본 정보 (이름, 나이, 외모, 성격)
- 배경 스토리
- 다른 인물과의 관계
- 능력/스킬 (판타지/무협 등)
- 성장 곡선 / 변화 예정 사항

## 6.4 AI 보조 기능 (AI Assistant)

### 6.4.1 줄거리 설정

**대화 방식:**
- 챗봇 형태 자유 대화
- AI 단계별 질문 → 사용자 선택/입력
- 둘 다 지원

**결과물 (단계적 생성):**
- 전체 스토리 시놉시스
- 권별 줄거리
- 화별 줄거리 (세부 플롯)

### 6.4.2 인물 설정

AI가 질문을 통해 인물 설정을 구체화하고 카드 자동 생성

### 6.4.3 글쓰기 보조

**기본 설정:**
- 1화당 목표 분량: 5,000자
- 1권 = 25화 구성

**글쓰기 모드 (사용자 선택):**
- AI가 전체 초안 작성 → 사용자가 수정
- 사용자가 작성 → AI가 교정/보완 제안
- AI와 사용자가 문단별로 협업

**컨펌 방식:**
- 1화 단위 컨펌
- 컨펌 전까지 언제든 수정 가능
- AI는 사용자 수정 요청 항상 수락

### 6.4.4 실시간 요약 패널

AI가 자동으로 업데이트하는 정보:
- 현재 진행 상황
- 등장인물 현재 상태
- 최근 화 요약 (AI 참조용)

### 6.4.5 설정 오류 감지

**1차 개발:** 단순 알림
- 예: "3화에서 A가 죽었는데 5화에 등장합니다"

**2차 전환 (조건부):** API 비용 과다 시 사용자 직접 관리로 전환

## 6.5 내보내기 및 호환성 (Export & Copy)

| 기능 | 설명 |
| --- | --- |
| 스마트 복사 (Plain) | 모든 서식 제거한 순수 텍스트 |
| 스마트 복사 (Structure) | 씬 구분, 지문, 대사 줄바꿈 구분 (HWP에 최적화) |
| 파일 내보내기 | txt, docx, PDF, 플랫폼별 포맷 |
| 백업 | 전체 원고 .txt 일괄 다운로드 |

# 9. MVP 및 개발 로드맵

2. 에디터 + 자동 저장

4. 좌측 패널 탭 전환 (AI 대화창 제외)
